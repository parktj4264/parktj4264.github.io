반도체 공정 데이터 A/B 테스트에서 Glass’s Delta를 사용하는 이유

기존의 Pooled Standard Deviation 대신 Reference의 Standard Deviation만 사용하는 이유와 그 심층적 의미

1. 서론: 반도체 공정에서의 A/B 테스트와 데이터의 특성

  • A/B 테스트의 목적: 새로운 공정 조건(Target)이 기존 양산 조건(Reference) 대비 얼마나 개선되었는지(혹은 산포가 줄었는지) 확인.
  • 반도체 데이터의 고유한 특징:
    • 양산(Reference) 데이터는 오랜 기간 축적되어 매우 안정적이고 표본 수가 많음.
    • 평가 및 실험(Target) 데이터는 표본 수가 적고 변동성이 클 수 있음.
  • 문제 제기: 일반적으로 두 그룹의 차이를 비교할 때 보통 Pooled Standard Deviation (합동 표준편차)를 많이 사용하지만, 왜 우리 분석에서는 Reference의 Standard Deviation만 사용하는 Glass’s Delta를 쓸까?

2. 기존 방식(Cohen’s d)의 한계점

  • Cohen’s d란?: 두 그룹의 평균 차이를 두 집단의 분산이 같다는 가정 하에 Pooled SD로 나눈 Effect Size 지표.
  • 단점 및 한계:
    • 두 그룹의 샘플 사이즈나 분산이 크게 다를 경우(반도체에서는 흔한 일), Pooled SD는 왜곡될 수 있음.
    • 실험군(Target)의 이상치(Outlier)나 큰 산포에 의해 영향을 크게 받아, 실제보다 Effect Size가 과소/과대 추정될 위험 존재.

3. 왜 Reference(대조군)의 Standard Deviation만 사용하는가? (Glass’s Delta)

  • Glass’s Delta의 정의: \(\Delta = \frac{\bar{X}_{target} - \bar{X}_{ref}}{SD_{ref}}\)
  • Reference의 안정성: 양산 기준 조건인 Reference의 산포(\(SD_{ref}\))는 이미 오랜 시간 검증된(안정화된) 자연스러운 변동성을 대변함.
  • 비교의 기준 확립: 새로운 조건(Target)의 성과를 “우리가 이미 알고 있는 흔들림(Reference SD)”을 잣대로 평가하는 것이 공학적으로 훨씬 합리적임.
  • Target 분산의 영향 배제: 실험군의 조건이 불안정해서 커진 분산이 잣대(분모)에 섞여 들어가는 현상을 막아줌.

4. 직관적인 예시와 해석 (데이터 관점)

  • (여기에 간단한 수치 예제나 가상의 Plot 결과를 묘사하여 추가할 예정)
  • 예: Target 그룹은 평균이 이동했지만 산포가 엄청나게 커진 경우, Pooled SD를 쓰면 평균 이동 효과가 작아 보이지만, Ref SD를 쓰면 기준 대비 얼마나 벗어났는지 정확히 캐치할 수 있음.

5. 결론 및 요약

  • 반도체 데이터 분석 시, 두 그룹의 분산이 다를 가능성이 높고 Reference 그룹의 대표성이 월등히 높기 때문에 Glass’s Delta가 유리함.
  • 결국 통계 수식은 ’우리가 무엇을 기준으로 변화를 감지할 것인가’라는 철학과 닿아 있음.
  • 현업 적용 포인트: 새로운 공정이나 설비 개조 후 데이터 검증 시 이 관점을 가지고 데이터 분석을 진행하자.
